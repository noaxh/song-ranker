1. Cloud backup + multi-device sync. Right now everything lives in one browser's localStorage. Clear cache or switch devices and the entire rating history (hours of work) is gone. This is the app's biggest structural risk. Minimum: surface explicit JSON export/import as a real backup workflow (the merge plumbing already exists in store.js). Bigger: keyed cloud store (Supabase or a tiny backend, indexed by Spotify user id) for auto-sync across phone/desktop. Highest impact because it protects the thing users invest the most in.

2. Export rankings back to Spotify as playlists. The app pulls FROM Spotify but the output (your S-tier, a group's ranked order, top 100) never leaves the app. Add "Create Spotify playlist from this tier/group/leaderboard," optionally kept in sync. Turns a private scoreboard into something you actually listen to. The API layer already does playlist reads, so this is create + add-tracks on top. Closes the loop, daily utility.

3. Smart rating sessions + engine reconciliation. A guided "rate next" queue that surfaces the highest-value comparisons first (your Elo code already knows which songs are most uncertain, low duel count), plus a "resolve disagreements" mode that flags songs where Quick Rank and Face-off disagree (the two-engine conflict I found earlier). Gamify with streaks/daily goals. Drives engagement and makes the ranking converge faster and more trustworthy. Optional add-on: enrich with Spotify audio features (energy/tempo/mood) for "By Mood" grouping and taste insights, but caveat: Spotify restricted the audio-features endpoint for newer apps in late 2024, so verify access before committing to that part; the play-count data you already collect is a safe fallback ("most-played but unrated", "highest-played vs highest-rated gap").

My ranking: #1 first (data safety is existential), then #2 (utility), then #3 (depth/engagement). Want me to spec or build any of them?
